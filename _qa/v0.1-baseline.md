# v0.1 — TTFA baseline (Mac Air M4)

Date: 2026-06-03
Binary: `zig-out/bin/agent-tts` (Zig 0.16, ReleaseFast, 415KB, arm64 Mach-O)
Voice: `Luciana` (plain, no Premium installed)
Command: `say -v Luciana -r <rate> <text>` via `std.process.spawn`

## Spawn latency (5 cold runs, rate 180, short text)

| Run | spawn (ms) |
|-----|-----------|
| 1   | 0.8       |
| 2   | 0.8       |
| 3   | 0.5       |
| 4   | 0.8       |
| 5   | 0.7       |

Mean: **0.72ms**. Median: 0.8ms.

> Metric = time between process start and `std.process.spawn` returning. **Not real TTFA** (time-to-first-audio) — does not capture the time between `say` exec and the first audible sample. Real TTFA needs `dtruss -t write` on `say` + audio capture (v0.2+).

## Rate sweep (Luciana, same phrase, ~10 words)

| Rate (wpm) | Total (ms) |
|------------|------------|
| 180        | 3953       |
| 220        | 4103 (different phrase) |
| 360        | 3862       |
| 450        | 3164       |
| 540        | 2651       |
| 600        | 2428       |

Total-time reduction drops roughly linear up to 540, then plateaus. Above 600 intelligibility collapses (not formally measured — judged by ear).

## Chosen default: 330 wpm

Decision made by ear (author, 2026-06-03):

- 180 = too slow for an agent
- 430 = intelligible but dry/rushed on long sentences
- 330 = sweet spot — human, fast, sustains ~80-word sentences without losing words

## v0.1 → v0.2 gaps

- Real TTFA (audio-sample detection) not yet measured
- No pre-warm — the first call of the day pays the Speech Synthesis Manager cold cost
- No daemon = every call pays fork+exec of `agent-tts` + fork+exec of `say`
- No queue = parallel calls overlap (bad UX)

All of these become v0.2 targets.
