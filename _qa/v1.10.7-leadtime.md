## What shipped

- **`src/ipc.zig`** — 8-field ENQUEUE with optional tune triplet (`length:noise:noise_w`, `-` for unset, empty for all-default). `Message` gained `length_scale`/`noise_scale`/`noise_w`. `parseRequest` disambiguates 7- vs 8-field via `looksLikeTuneTriplet`. Added 8 new tests (parser, formatter, round-trip, backward-compat).
- **`src/queue.zig`** — 3 new REAL columns (`length_scale`, `noise_scale`, `noise_w`); idempotent ALTER TABLE migration; NULL-aware read; `push` + `tryClaimNext` + `replay` thread them through.
- **`src/piper.zig`** — `synthToSamplesTuned(arena, text, length_scale, noise_scale, noise_w)`. `synthToSamplesScaled` shimmed to `synthToSamplesTuned(... -1, -1)`. Env-var fallthrough via new `envFloat` helper (`AGENT_TTS_PIPER_LENGTH_SCALE`/`_NOISE_SCALE`/`_NOISE_W`). `MultiPiperEngine.synthLangTuned` dispatches Pt/En per route; stub mirrors signature for `-Dwith-piper=false`.
- **`src/daemon.zig`** — `runPiperSingle` + `synthWorker` thread item knobs through; logs `[worker] piper id=N length_scale=… noise_scale=… noise_w=…` whenever any override is set; SSML path still uses `synthToSamplesScaled` (one knob only).
- **`src/client.zig`** — `--length-scale`/`--noise-scale`/`--noise-w` flags with range validation. New `enqueueLineTuned`; `enqueueLineSsml` shimmed through it. CLI now writes via `ipc.encodeEnqueue` (single source of truth).
- **`src/mcp.zig`** — `say` schema gains 3 optional number params with range gates. New `synth_voice_test(text, length_scale, noise_scale, noise_w)` tool returns enqueue id + parsed knobs. VERSION→`1.10.7`. Total: 11 tools. `jsonNumberToF32` helper for both integer + float JSON inputs.
- **`src/main.zig` + `build.zig.zon`** — VERSION `1.10.7`. Main HELP gains the 3 flags.
- **Docs** — `changelog.md` v1.10.7 entry; `motor.md` Tuning Piper per call section + recommended Faber profiles + SSML interaction note; `mcp.md` 11 tools + per-call knob table on `say`; `roadmap.md` + `whats-next.md` rows.
