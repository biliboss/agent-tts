## Why agent-tts

Your agents already write, build, deploy, and review. They should also **talk** — without stealing focus, without a Python venv, without paying ElevenLabs per token.

```bash
agent-tts "Build green in 12 seconds."
agent-tts --engine piper "Deploy concluído, três warnings."
agent-tts queue       # list what is pending
agent-tts skip        # kill current
```

| | agent-tts | macOS `say` | Piper (Python) | ElevenLabs |
|---|---|---|---|---|
| Warm synth latency | **~91ms** | n/a | 200-300ms | 200-800ms + network |
| Cold daemon boot | **~720ms** | 0 per call | 1-3s | 0 |
| Persistent queue | ✅ SQLite WAL | ❌ | ❌ | ❌ |
| Skip / clear / list | ✅ | ❌ | ❌ | n/a |
| Auto-start | ✅ launchd | n/a | manual | n/a |
| Single binary | ✅ ~975 KB | n/a | ❌ Python venv | ❌ |
| Offline | ✅ | ✅ | ✅ | ❌ |
| Cost per message | $0 | $0 | $0 | $$$ |

## One KPI: time-to-first-audio

Everything is benchmarked against the gap between calling `agent-tts "text"` and the first audible sample.

| Scenario | Target | Measured |
|---|---|---|
| Warm daemon, voice loaded | **< 300ms** | round-trip 0.1ms + synth 91ms |
| First call after boot (cold) | < 800ms | pre-warm 280ms + zaudio 79ms + piper 373ms |
| Long text — first chunk | < 400ms | (v1.1) |

## Designed for AI agents

- **Stable line protocol** over UNIX socket — easy to wrap from any language
- **Sub-millisecond round-trip ACK** — the agent moves on while the voice plays
- **Crash-safe queue** in SQLite WAL — texts survive a daemon restart
- **No paywall, no quota** — runs entirely on your Mac

## Locked constraints

- **Platform**: Apple Silicon macOS in v1.0 (Linux v1.1+)
- **Footprint**: single ~975 KB binary, no embedded voice model
- **Single binary**: CLI and daemon share the executable
- **Pt-BR human cadence**: cardinals, abbreviations, and punctuation pauses preprocessed

> Living document. Every decision is re-evaluated if measured TTFA drifts off target. PRs welcome at [biliboss/agent-tts](https://github.com/biliboss/agent-tts).
