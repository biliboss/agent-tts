## North star

**Time-to-first-audio (TTFA)**: the gap between calling `agent-tts "text"` and the first audible sample at the speaker.

v1.0 target on Mac Air M4: **< 300ms** with a warm daemon cache. Every stack choice is justified against that number.

| Scenario | TTFA target |
|---------|-----------|
| Warm daemon, voice already loaded | < 300ms |
| First call after boot (cold) | < 800ms |
| Long text (>500 chars) — first chunk | < 400ms |

## Why it exists

Agents (Claude Code, local CI scripts, watchers) need to speak updates without stealing visual focus. Pop-ups interrupt, terminal scroll is silent, notifications disappear. Voice fills the whole audio channel and keeps your hands free.

Planned commands:

```bash
agent-tts "Build green in 12s."
agent-tts --rate 200 "Fast text."
echo "stdin too" | agent-tts -
agent-tts queue           # list pending
agent-tts skip            # skip current item
```

## Locked constraints

- **Platform**: Mac Air M4 only in v1.0 (Linux v1.1+)
- **SSD**: binary < 2MB, no embedded model
- **Single binary**: CLI and daemon share the executable
- **Human Pt-BR**: natural voice, default cadence 180wpm, pauses driven by punctuation

> Living document. Every decision is re-evaluated if measured TTFA drifts off target.
